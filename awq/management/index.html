Para ingestão real: execute localmente (npm run dev) ou migre para Vercel + armazenamento externo (Vercel Blob / Postgres).

§5 · Regras de Governança (Bloqueios Estruturais)

ATIVO
Sem valor financeiro sem FinancialMetric<T>
Toda métrica financeira exibida em página central deve ser um objeto FinancialMetric<T> com source_type declarado. Nunca exibir número bruto sem proveniência.
lib/financial-metric-registry.ts
ATIVO
Sem importação direta de awq-group-data em pages/
Pages devem importar de lib/awq-derived-metrics.ts (camada canônica). Nenhuma página pode importar lib/awq-group-data.ts diretamente.
Verificação: grep -r 'from "@/lib/awq-group-data"' app/
ATIVO
Sem novos arrays financeiros hardcoded em lib/
Antes de adicionar qualquer nova constante de dados financeiros, registre em lib/snapshot-registry.ts. Se não há justificativa para um entry no registry, não adicione o hardcode.
lib/snapshot-registry.ts
ATIVO
Fontes reais exclusivamente via buildFinancialQuery()
Dados cash-basis (extratos bancários) devem ser consumidos exclusivamente por buildFinancialQuery() em lib/financial-query.ts. Nenhuma leitura direta de financial-db.ts em pages.
lib/financial-metric-query.ts (único entry point para pages)
ATIVO
Empty state honesto — nunca inventar valores
Quando não há dados (emptyMetric), o UI deve exibir '—' ou MetricEmpty. Nunca fallback silencioso para 0 ou para snapshot sem badge visível.
metricDisplay() em financial-metric-registry.ts + MetricEmpty componente
ATIVO
Snapshot sempre com badge visível
Qualquer métrica com source_type=snapshot ou fallback deve exibir MetricSourceBadge e MetricDetail no componente de card.
components/MetricSourceBadge.tsx
